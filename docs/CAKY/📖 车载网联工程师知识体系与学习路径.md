 ### 📌 模块 2：T-BOX 硬件与嵌入式（职责 3-5）

 #### 核心知识点

 ```
   硬件组成：
   ├── 通信模组：4G Cat.4/Cat.6/5G · GNSS · WiFi · 蓝牙
   ├── 主控芯片：ARM Cortex-A/R/M 系列
   ├── 存储：eMMC · UFS · DDR
   ├── 接口：CAN/CAN FD · Ethernet · USB · SIM
   └── 安全：HSM · 安全启动 · 加密存储

   软件栈：
   ├── OS：Linux/QNX/RTOS
   ├── 中间件：AUTOSAR CP/AP
   ├── 协议栈：TCP/IP · MQTT · HTTP/2 · Some/IP
   └── 应用：远程控车 · OTA · 诊断 · 数据采集
 ```

 #### 学习资源

| 类型      | 资源                                          | 说明            |
|---|---|---|
| 📚 书籍   | 《汽车电子硬件设计》《嵌入式系统架构》        | 理解硬件选型    |
| 📄 规格书 | 高通 9205/9206、移远 AG55x、广和通 L860       | 主流 T-BOX 方案 |
| 🏭 供应商 | 华为/移远/广和通/美格智能官网技术文档         | 了解产品参数    |

   ├── 服务层：
   |   ├── 车辆管理：车辆档案 · 状态监控 · 故障诊断

 #### TSP 招标关键评审点（你的核心工作）

 ```
   □ 技术标：
     ├── 架构设计：是否支持百万级车辆并发
     ├── 安全合规：等保三级、数据出境、隐私保护
     ├── 高可用：SLA≥99.9%、灾备方案、容灾演练
     └── 扩展性：微服务化、弹性伸缩、多租户

     ├── 团队资质：汽车行业案例、CMMI/ASPICE 认证
     ├── 交付计划：里程碑、人员投入、风险预案
     └── 运维服务：7×24 支持、响应时间、驻场要求

   □ 演示验证：
     ├── 功能演示：远程控制、OTA、诊断
     ├── 性能压测：并发用户数、指令响应时间
     └── 安全测试：渗透测试报告、漏洞扫描
 ```

 ### 📌 模块 6：功能安全与网络安全（职责 3-9）

 #### 功能安全（ISO 26262）

 ```
   ASIL 等级：A(最低) → B → C → D(最高)
   T-BOX 典型等级：ASIL B

   关键产出：
   ├── HARA 分析（危害分析与风险评估）
   ├── 安全目标（Safety Goal）
   ├── 安全需求（Safety Requirement）
   ├── 安全架构（Safety Architecture）
   └── 安全案例（Safety Case）
 ```

 #### 网络安全（ISO/SAE 21434）

 ```
   TARA 分析：
   ├── 资产识别
   ├── 威胁分析
   ├── 风险评估
   └── 风险处置

   关键要求：
   ├── 安全启动
   ├── 安全通信（TLS 双向认证）
   ├── 安全存储（密钥管理）
   ├── 安全 OTA（签名验证）
   └── 渗透测试（年度）
 ```

 #### 学习资源

| 类型    | 资源                                    | 说明         |
|---|---|---|
| 📚 书籍 | 《汽车功能安全 ISO 26262 实战指南》     | 入门必读     |
| 📄 标准 | ISO 26262、ISO/SAE 21434、GB/T 自动驾驶 | 精读核心章节 |
| 🎓 培训 | TUV 南德/莱茵功能安全认证培训           | 考取证书     |
| 🏢 咨询 | 聘请第三方做 TARA 分析（项目初期）      | 实战学习     |
